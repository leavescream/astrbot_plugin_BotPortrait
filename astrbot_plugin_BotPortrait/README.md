# BotPortrait - AI 自画像 🎨

> 每次对话都带着自己样子的 AI 自画像插件。

**BotPortrait** 是 AstrBot 的一款生活化自画像插件。它的核心理念是：**让 AI 角色拥有稳定的视觉形象，同时根据对话内容自然地展现动作和心情**。

---

## 🎯 核心优势

### 🧱 固定关键词 → 角色形象稳定
- 每个人格可配置**固定正面提示词**
- 每次生图，这些关键词**始终注入 prompt 头部**，确保角色形象不会漂移
- 例如：`"白发蓝瞳、机械身躯、黑色发带"` —— 永远不会丢失
（手打添加：要用英文提示词）

### 👤 按人格区分配置
- 基于 `personality_key` 区分不同 AI 人格
- 每人格可独立设置：触发概率、固定提示词、穿搭/发型开关、引擎参数（模型、LoRA、尺寸、采样器）
- 自动匹配当前会话人格，匹配失败则视为**无模型状态，不会生图**
（手打添加：就是创建人格时的人格ID，注意要区分大小写的）

### 🧠 三层 prompt 构建逻辑
```
固定关键词（core_prompt） ← 稳定角色形象
        +
穿搭 + 发型标签（每日轮换） ← 生活感、差异化
        +
动作/心情标签（LLM 提取）  ← 响应对话上下文
```

**关键约定**：子 LLM 在提取动作/心情标签时，**只生成动作、表情、镜头语言相关的关键词**，不会生成任何外观、服装、发型类关键词。外观由固定关键词和每日穿搭/发型系统负责，分工明确。

## ✨ 功能一览

### 📸 自动生图（回复触发）
- 每次回复消息后，按**人格专属概率**自动触发生图
- 支持**流式输出**兼容——流式结束后才触发，不影响文字回复速度
- **重复抑制**：本轮对话已通过 `take_selfie` 出图则跳过自动触发
- **互斥锁**：防止并发生图冲突
（手打添加：防互斥就是，你也可以直接让ai使用工具生图，这时候不会触发自动生图）

### 🧥 每日穿搭系统
- 从穿搭池中每日随机挑选一套穿搭
- 子 LLM 自动生成中文描述 + 英文 SD tags，注入 system prompt
- 支持**刷新时间自定义**（默认每天 00:00）
- 支持手动重选、指定风格、查看库
- **3天重复避免**：追踪近期记录

### 💇 每日发型系统
- 与穿搭独立随机，每日从发型池中挑选
- 同样支持刷新时间、手动重选、指定

### 🤳 AI 主动自拍（take_selfie）
- 告诉 AI "拍一张"、"摆个姿势"之类的话，AI 会主动使用工具进行生图。
（手打添加：这样可以更主动的控制生图关键词，不过穿搭与发型开着的话还是会加入的）

## 🔧 安装 & 配置

### 安装
1. 将本插件目录放入 AstrBot 的 `plugins` 目录
2. 在插件配置页面填写必要参数
3. 配置页面中必须生成人格模板，没有可用的默认配置。

### 全局配置

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `全局开关` | bool | true | 总开关 |
| `ComfyUI API 地址` | string | `http://127.0.0.1:8188` | ComfyUI 服务地址 |
| `生成超时（秒）` | int | 30 | ComfyUI 等待超时 |
| `自动清理 ComfyUI output` | bool | false | 每次生图后清理旧图 |
| `每日穿搭风格池` | list | 11种风格 | 格式 `款式名：该款式描述` |
| `每日发型风格池` | list | 21种发型 | 格式 `款式名：该款式描述` |
| **`每日穿搭发型随机时间`** | string | `00:00` | HH:MM 格式，到点才重选 |
| `自画像分镜规则（翻译说明书）` | text | - | 子 LLM 分镜导演规则书 |

### 人格配置（template_list）

每个人格可独立设置：

| 配置项 | 类型 | 说明 |
|--------|------|------|
| `personality_key` | string | 人格标识，如 `Echo` |
| `触发概率` | float | 0.0~1.0，每次回复触发生图的概率 |
| `固定正面提示词` | text | **每次生图固定注入 prompt 头部**，用于稳定角色形象 |
| `固定负面提示词` | text | 每次生图固定注入的负面词 |
| `每日穿搭` | bool | 是否启用每日穿搭 |
| `每日发型` | bool | 是否启用每日发型 |
| `启用自动储存图片` | bool | 是否自动保存图片 |
| `自动储存图片路径` | string | 图片保存目录 |
| `引擎参数` | object | 工作流路径、模型、LoRA、尺寸、采样器等 |

## 🔄 完整工作流程

```
用户消息
  │
  ▼
on_llm_request ──注入 <character_state>（今日穿搭 + 发型）
  │
  ▼
LLM 回复（流式）
  │
  ▼
on_decorating_result (STREAMING_FINISH)
  │
  ├─ 已由 take_selfie 出图？→ 跳过
  ├─ 概率未命中？→ 跳过
  ├─ 正在生成中？→ 跳过
  │
  ▼
子 LLM 分镜导演 ← 提取回复内容
  │  ├─ action_hint（动作/表情描述） ← 只提取动作心情，不碰外观
  │  └─ SD tags
  │
  ▼
拼接最终 prompt:
  [固定正面提示词] + [穿搭 tags] + [发型 tags] + [动作心情 tags]
  │
  ▼
ComfyUI 渲染 → 输出图片 → 发送到对话
```
